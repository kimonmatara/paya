What’s New

Version 0.4.2

General Changes

  • paya.plugtypes.matrix.Matrix() and paya.datatypes.matrix.Matrix() escape into simpler implementations, with sparser DG graphs, when all compensation keyword arguments evaluate as False

Version 0.4.1

General Changes

  • Smarter defaults handling on paya.nodetypes.ikHandle.IkHandle.create()

  • Added IK Handles section to documentation

  • Removed Chain() subclasses to simplify

Bugs Fixed

  • createFromCurve() fails with error about missing ‘override’ object

  • Missing createIkHandle()

  • orient() does not return self

Version 0.4

General Changes

  • Naming arguments on createNode() and elsewhere have been simplified, with many moved into paya.config instead.

Additions to Modules

  • Added paya.config

  • Added paya.lib.skel, including Chain and subclasses

Additions to Data Types

  • On Vector: rotateByAxisAngle() asScaleMatrix()

  • On Matrix: decomposeAndApply()

Additions / Changes to Plug Types

  • On Attribute: release()

  • On Compound: release() splitInputs()

  • On Vector: rotateByAxisAngle()

  • On Angle: set()

  • On EulerRotation: set() now auto-converts units when the value is an EulerRotation; added asRotateMatrix()

  • On Matrix: decomposeAndApply()

Additions to Node Types

  • On Transform: releaseSRT() getWorldMatrix() getWorldPosition() getRotateAxisMatrix()

  • On Joint: skinClusters() ikHandles() chainFromHere() getJointOrientMatrix()

  • On IkHandle: create() getEndJoint() getJointList() (overload for includeTip) chain() setTwistVectors() setPolePoint()

  • On NurbsCurve: takeClosestPoint() takePointAtParam() takeParamAtFraction() takePointAtFraction() distributePoints()

Additions to General Classes

  • On Chain: getFromStartEnd() getFromRoot() createFromMatrices() createFromPoints() createFromCurve() createFromCurve() orient() bones() vectors() points() contiguous() roots() length() insertJointsAtRatios() skinClusters() ikHandles() downAxis() duplicate() rename() compose() explode() appendChain() getPolePoint() autoPreferredAngle() reset() createIkHandle() createIkHandles()

  • On Bone: insertJoints()

Version 0.3

General Changes

The abstract plug tree has been reworked to make names more intuitive and to better support angles and euler rotations.

Bugs Fixed

  • Contextual naming misapplied when createNode() is used for shapes

  • Some universal operators don’t work properly when called on paya.datatypes.point.Point rather than paya.datatypes.vector.Vector

Additions to Modules

  • On paya.lib.mathops: createMatrix() asValue()

Additions to Node Types

  • On FourByFourMatrix: getAxis() getX() getY() getZ() getTranslate() translate/t x y z

  • On Shape: createNode()

Additions to Plug Types

  • On Attribute: plugType()

  • On Angle: get()

  • On Compound: __iter__()

  • On Enum: put()

  • On EulerRotation: get() isRotateChannel() asQuaternion() asRotateMatrix()

  • On Math1D: sqrt() __neg__() abs() cycle() remap() clamp() minClamp() min() eq() ne() gt() ge() lt() le() choose() blend() unaryExpr() degrees() radians() cos() sin() tan() acos() asin() atan() atTime()

  • On Math3D: get() blend()

  • On Matrix: createLocator() pick() inverse() asOffset() getAxis() getX() getY() getZ() getTranslate() x y z translate/t transpose() decompose() hold() asEulerRotation() asQuaternion()

  • On Quaternion: inverse() normal() conjugate() blend() asEulerRotation() asAxisAngle() asRotateMatrix()

  • On String: put()

  • On Vector: dot() length() normal() cross() angle() asTranslateMatrix() asScaleMatrix() asEulerRotation()

Additions to Data Types

  • On EulerRotation: __add__() __radd__() __sub__() __mul__() __rmul__() __truediv__() __rtruediv__() __pow__() __rpow__()

  • On Matrix: createLocator() decompose() pick() asOffset() hold() getAxis() setAxis() getX() setX() getY() setY() getZ() setZ() getTranslate() setTranslate() x y z translate / t

  • On Point: __add__() __radd__() __sub__() __rsub__() __mul__() __rmul__() __truediv__() __rtruediv__() __pow__() __rpow__()

  • On Vector: blend() dot() cross() angle()

Version 0.2

Additions

  • A wide range of maths operators, spread across paya.plugtypes and paya.datatypes, with support for bidirectional mixing of values and plugs

  • Added pick() and asOffset() to AttributeMath16D

  • Added pick() and asOffset() to Matrix

  • get() returns more useful types

  • The double-chevron operator >> now supports value-setting

  • Contextual name management with paya.lib.names.Name and pipeline suffixes

  • Dodge unit conversion nodes with paya.lib.mathops.NativeUnits and paya.lib.mathops.nativeUnits()

Version 0.1

Additions

  • Patching engine

  • paya.runtime interface
