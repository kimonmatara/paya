IK Handles

Creating IK Handles

Paya provides a few different constructors in addition to the standard PyMEL ikHandle() command. The first is paya.runtime.nodes.IkHandle.create(), which takes the same arguments and supports managed names:

with r.Name('leg'):
    r.nodes.IkHandle.create(sj='joint1', ee='joint5')

# leg_KHDL

You can also call createIkHandle() on Chain instances. This defaults to ikSCsolver for two-joint chains and ikRPsolver for longer ones:

chain = r.PyNode('joint1').chainFromHere()
chain.createIkHandle()

A ‘plural’ form, createIkHandles() can be used to create one SC handle for each bone in a chain. The IK handles will be numbered; prefixes and padding can be specified via Name blocks:

with r.Name('leg', padding=2):
    print(chain.createIkHandles())

# [nt.IkHandle('leg_001_KHDL'), nt.IkHandle('leg_002_KHDL')]

Preventing Lockout on Inline Chains

Use autoPreferredAngle() to prevent lockout when IK is applied to inline chains. The method requires an upAxis hint:

chain = r.PyNode('_1_JOIN').chainFromHere()
chain.autoPreferredAngle('x')
ikh = chain.createIkHandle()

Pole Vectors

The setPolePoint() method on IkHandle is an alternative to poleVectorConstraint() that works with point attributes and implements maintainOffset:

loc, ikh = r.ls(sl=1)
ikh.setPolePoint(loc.attr('worldPosition'), maintainOffset=True)

To get non-jumping pole vector defaults for complex non-planar chains, use getPolePoint():

ikh = r.ls(sl=1)[0]
chain = ikh.chain()

userLoc = r.nodes.Locator.createNode().getParent()
userLoc.attr('translate').set(chain.getPolePoint(distance=17))
polePoint = userLoc.attr('worldPosition')

ikh.setPolePoint(polePoint)

Spline Twist

Use setTwistVectors() on IkHandle to quickly enable and configure start / end twist on spline handles. This pairs well with axis properties on Paya Matrix attributes:

ikh = r.PyNode('KHDL')

cube1 = r.PyNode('pCube1')
cube2= r.PyNode('pCube2')

ikh.setTwistVectors(cube1.attr('wm').y, cube2.attr('wm').y, 'x')
