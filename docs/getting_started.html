Quick Start

Requirements and Installation

Paya requires Python 3.x and Maya 2022 or higher. To install it, download the package from here, unzip, move into your scripts directory and relaunch Maya.

Windows

C:\Users\<username>\My Documents\Maya\<version>\scripts

Mac

/Users/<username>/Library/Preferences/Autodesk/Maya/<version>/scripts

Linux

~<username>/Maya/<version>/scripts

Importing and Patching

Paya extends PyMEL using a fast runtime patch. The patch is applied when you first import paya.runtime:

import pymel.core as p
import paya.runtime as r
# PyMEL patched successfully.

cam = p.PyNode('persp')
print(cam.__class__)
# <class 'paya.runtime.nodes.Transform'>

Notice the inspected class of the camera instance; PyMEL will continue to return custom Paya classes instead of its own until stop() is called:

r.stop()
# PyMEL unpatched successfully.

cam = p.PyNode('persp')
print(cam.__class__)
# <class 'pymel.core.nodetypes.Transform'>

At this point, PyMEL has been restored to its ‘factory’ state. If you want to re-apply the patch, importing paya.runtime won’t work a second time. Instead, you must call start():

r.start()
# PyMEL patched successfully.

Using the Runtime Interface

In previous examples, we used paya.runtime merely to manage patching, with PyMEL imported via the customary pymel.core. In fact paya.runtime includes the entire pymel.core namespace, and can therefore be used as a drop-in replacement:

import paya.runtime as r
# PyMEL patched successfully.

cameras = r.ls(type='camera') # etc

Using paya.runtime in this way gives you easy access to top-level Paya-only tools such as paya.lib.controls.createControl() and paya.lib.mathops.createMatrix(); it also exposes the nodes, plugs, comps and data class pools, which can be used to call special constructors on custom Paya classes:

import paya.runtime as r

xform = r.nodes.Transform.create(dagPath='|rig_XFRM|controls_XFRM') # Paya only

Importing paya.runtime instead of pymel.core also helps to signal that a code block is designed for ‘patched’ PyMEL.

Discovering Functionality

For a structured, workflow-oriented introduction to Paya, visit the Rigging sections of this documentation. If you’d rather just shop for methods, call help() on any PyMEL instance inside Maya:

joint = r.PyNode('joint')
help(joint.attr('r')) # list methods on rotational attributes

This will print out the same information as found inside Types. If you’re looking for a method and you’re not sure what it’s called, use dir() on a class:

jointClass = joint.__class__ # type(joint) also works

for x in dir(jointClass):
    if 'chain' in x.lower(): print x

# chainFromHere

help(joint.chainFromHere) # Now that you've found the method, get the docs
