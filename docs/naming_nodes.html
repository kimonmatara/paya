Managing Node Names

Node names in typical Maya pipelines follow the convention <several_prefixes>_<type>, where <several_prefixes> are contributed by various functions that take them as arguments, and <type> is something like GRP for groups, CTRL for controls, and so on. Paya gives you a couple ways to generate such names more easily.

Automatic Suffixes

All bundled Paya node constructors apply type suffixes by default. For example:

r.nodes.Joint.createNode(name='eyeball')
# Result: eyeball_JOIN

Suffixes are mapped to node types, and applied according to this rule:

  • If the node is a transform, then:

    • If the node has a shape, the lookup is the shape’s node type

    • Otherwise, the lookup is the transform’s node type

  • Otherwise:

    • If the node is a shape, no suffix is applied

    • Otherwise, the lookup is the shape’s node type

Suffixes can be overriden, or disabled, on a per-node basis like this:

r.nodes.Joint.createNode(name='eyeball', suffix=False) # disable
# Result: eyeball

r.nodes.Joint.createNode(name='eyeball', suffix='JNT') # override
# Result: eyeball_JNT

Default Paya suffixes are derived from the ‘tags’ in the Maya Module Index. To redefine or disable them globally, see Package Configuration.

Implicit Naming

Paya’s Name context manager allows you to build up prefixes implicitly, instead of passing them around as function arguments. For example, here’s a typical naming pattern:

def makeLimb(prefix):
    r.group(empty=True, name='{}_GRP'.format(prefix))

for side in 'LR':
    for typ in ['arm', 'leg']:
        makeLimb('{}_{}'.format(side))

# Produces L_arm_GRP, R_arm_GRP, L_leg_GRP, R_leg_GRP

Here’s an alternative using Name:

def makeLimb(): # no prefix argument
    return r.nodes.Transform.createNode()

for side in 'LR':
    for typ in ['arm', 'leg']:
        with r.Name(side, typ):
            makeLimb()

# Produces L_arm_GRP, R_arm_GRP, L_leg_GRP, R_leg_GRP

This also works with maths operations:

with r.Name('mech'):
    loc1.t ^ loc2.wm

# Produces a pointMatrixMult node named mech_DPMM

Name can be nested arbitrarily deeply. Notice that the blocks cut through function calling scopes. This is an example of aspect-oriented programming. It declutters your code and makes it easier to refactor. See the documentation for Name, and constructors like createNode(), for options to change block naming behaviour.

Note

To use automatic suffixing and Name blocks in your code, you’ll have to create nodes using the constructors on paya.runtime, since the behaviour of ‘vanilla’ PyMEL or maya.cmds functions is unmodified:

with r.Name('L', 'eyeball'):
    r.nodes.Joint.createNode() # produces L_eyeball_inner_JOIN
    r.createNode('joint') # produces joint1

Alternatively, you can use makeName() to construct managed names and pass them along:

import paya.runtime as r
import maya.cmds as m

with r.Name('L', 'eyeball'):
    m.createNode('joint', name=r.nodes.Joint.makeName())
