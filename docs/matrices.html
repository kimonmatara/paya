Matrices

Relevant Classes

Static

Dynamic

paya.datatypes.matrix.Matrix

paya.plugtypes.matrix.Matrix

Multiplying

Static and dynamic matrices can be multiplied using the universal * operator. Recall from Operator Basics that at least one of the operands must be an instance of Attribute or Matrix:

driver1 = r.PyNode('locator1')
driver2 = r.PyNode('locator2')
driven = r.PyNode('pCube1')

matrix = driver1.attr('worldMatrix') * driver2.attr('worldMatrix')
matrix >>  driven.attr('offsetParentMatrix')

Inverting

Use the universal inverse() method to reverse a transformation:

matrix3 = matrix1 * matrix2
matrix1 = matrix3 * matrix2.inverse()

Applying

The most performant way in Maya to apply a matrix to a transform node is to set, or connect, the node’s offsetParentMatrix input. This also has the advantage of leaving the SRT channels free for editing:

driver = r.spaceLocator()
driven = r.polyCube(ch=False)[0]

driver.attr('worldMatrix') >> driven.attr('offsetParentMatrix')

To modify SRT channels selectively via decomposition, use the universal method decomposeAndApply(). By default this accounts for rotateAxis and, on joints only, jointOrient and inverseScale. In the following example rotateAxis compensation is disabled for a simpler network:

driver.attr('worldMatrix').decomposeAndApply(
    driven,
    translate=True,
    rotate=True,
    compensateRotateAxis=False
    )

Filtering

To quickly configure and connect a pickMatrix node, use pick(). This has also been implemented on data classes. Here’s a variant of the earlier selective-application example without matrix decomposition:

translationMatrix = locator1.attr('worldMatrix').pick(translate=True)
rotationMatrix = locator2.attr('worldMatrix').pick(rotate=True)
combinedMatrix = rotationMatrix * translationMatrix

combinedMatrix >> cube.attr('offsetParentMatrix')

Offsetting

Use asOffset() to preserve relative pose when applying a dynamic matrix to a transform. This is functionally the same as the maintainOffset option on Maya’s constraints:

initialPose = cube.getMatrix(worldSpace=True)
matrix = initialPose * loc.attr('worldMatrix').asOffset()

decomposition = matrix.decompose(ro=cube.attr('rotateOrder'))
decomposition['translate'] >> cube.attr('translate')
decomposition['rotate'] >> cube.attr('rotate')

Tip

  • When using a direct connection into offsetParentMatrix instead of decomposition, the above example won’t work as-is. This is because the driven object’s local transformation will be applied twice: once through initialPose, and again by the untouched SRT channels themselves. To get around this, you could reset the driven object’s local matrix to zero-out the SRT channels:

    loc.setMatrix(r.data.Matrix()) # identity matrix
matrix >> loc.attr('offsetParentMatrix')

    Alternatively, you could ‘dodge’ the SRT channels by incorporating an inversion of the driven object’s local matrix into the driver matrix:

    matrix = cube.getMatrix().inverse(
    ) * initialPose * loc.attr('worldMatrix').asOffset()

matrix >> cube.attr('offsetParentMatrix')

  • For polymorphic parity, asOffset has also been implemented on paya.datatypes.matrix.Matrix, where it merely returns an identity (‘null’) matrix.

  • On the dynamic class, the expansion of asOffset() is:

    self.get().inverse() * self

Creating

To create static or dynamic matrices from vectors and points, use the universal createMatrix() function. Available on paya.lib.mathops as well as the top-level paya.runtime interface, it’s designed to use as few nodes and operations as possible.

Orthogonal Construction

An orthogonal matrix can be constructed by passing two axis:vector pairs as four arguments to createMatrix(). The first pair can be thought of as the ‘aim’, with the second as the ‘up’. Translation can be specified via the translate / t keyword argument, and there are additional options to control the lengths of the second and third axis vectors:

aim = locator1.attr('worldPosition')
up = locator2.attr('worldPosition')

matrix = r.createMatrix('z', aim, 'x', up).pick(r=True)
matrix >> cube.attr('offsetParentMatrix')

Explicit Assembly

To specify a matrix completely, pass three axis:vector pairs (i.e. six arguments) to createMatrix() instead. If the input vectors are not orthogonal, the result will be a shear matrix:

cube = r.PyNode('pCube1')
zVector = r.PyNode('locator1').attr('worldPosition')
xVector = r.PyNode('locator2').attr('worldPosition')
yVector = [0, 1, 0]

matrix = r.createMatrix('z', zVector, 'x', xVector, 'y', yVector)

matrix >> cube.attr('offsetParentMatrix')

Quick Testing

To quickly test a matrix, call createLocator() / cl() on any static matrix or matrix attribute. This will create a locator to represent the matrix:

loc = matrix.createLocator()
