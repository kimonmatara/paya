Using the Runtime Interface

So far we’ve only used paya.runtime to manage patching, reverting to the customary pymel.core for calls to PyNode. In fact, while patching is active, paya.runtime serves the entire pymel.core namespace, and can be used as a drop-in replacement:

import paya.runtime as r
r.start()
cam = r.ls(type='camera')[0]

The pass-through does not work when patching is inactive. This is done on purpose, to ensure that Paya rigging scripts are not attempted in an unpatched state.

The paya.runtime object also gives you access to top-level interfaces and functions such as createMatrix(). (More on these later.)
