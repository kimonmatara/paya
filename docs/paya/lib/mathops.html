paya.lib.mathops¶

Defines supporting methods for maths rigging and, more broadly, mixed value / plug workflows.

getBaseVector(axis)¶

Given an axis letter (or ‘t’ for ‘translate’), returns an appropriate Vector or Point instance.

Parameters:axis (str) – the axis for which to retrieve a Vector or Point; must be one of: ‘x’, ‘y’, ‘z’, ‘-x’, ‘-y’, ‘-z’ or ‘t’ (for ‘translate’)
Returns:The vector or point.
Return type:Vector or Point
isTupleOrListOfScalarValues(x)¶
Parameters:x – the item to inspect
Returns:True if x is a tuple or list of scalar values, otherwise False.
Return type:bool
class NativeUnits¶

Context manager. Switches Maya to centimetres and radians.

nativeUnits(f)¶

Decorator version of NativeUnits.

info(item)¶

Returns a tuple of three members: - The item conformed to the highest-level type available - The item’s mathematical dimension (e.g. 16 for matrices) - True if the item is a plug, otherwise False

In short: item, dimension, isplug.

Parameters:item (any type) – the item to process
Returns:tuple
conform(x)¶

If x is a value, then:

  • If its dimension is 3, it’s returned as a Vector
  • If its dimension is 4, it’s returned as a Quaternion
  • If its dimension is 16, it’s returned as a Matrix
  • In all other cases, it’s returned as-is

If x is a plug, then it’s returned as an instance of the appropriate Attribute

Parameters:x – the item to conform
Returns:The conformed item.
Return type:Vector, Quaternion, Matrix or Attribute
multMatrices(*matrices)¶

Performs efficient multiplication of any combination of matrix values or plugs. Consecutive values are reduced and consecutive plugs are grouped into multMatrix nodes.

If any of the matrices are plugs, the return will also be a plug. Otherwise, it will be a value.

Parameters:*matrices (Matrix, AttributeMath16D, list or str) – the matrices to multiply (unpacked)
Returns:The matrix product.
Return type:Matrix or AttributeMath16D
